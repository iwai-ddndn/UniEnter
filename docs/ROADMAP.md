# UniEnter ロードマップ

現状(v0.1): コア機能は完成。M1〜M6 + アプリ別送信キー設定まで実装済み。

## リリースまでに必要(v1.0)

| 項目 | 内容 | 重要度 |
|---|---|---|
| Developer ID署名 + 公証(notarization) | App Store外配布の必須要件。未公証だとGatekeeperが起動をブロックし、一般ユーザーはほぼ脱落する。Apple Developer Program(年99ドル)への加入が必要 | ★★★ |
| 自動アップデート(Sparkle 2) | appcast.xml + EdDSA署名。新バージョンのお知らせ・リリースノート表示もこれで実現 | ★★★ |
| DMGパッケージング | 配布用DMG(アプリをApplicationsへドラッグする定番UI)。リリースはGitHub Releasesに置く | ★★★ |
| アプリアイコン | 現状SFシンボルのみ。⏎モチーフのアイコン制作 | ★★☆ |
| Chatworkの実機確認 | 対象アプリで唯一未検証 | ★★☆ |
| About画面 | バージョン表示・支援リンク・謝辞 | ★☆☆ |

## v1.x(フィードバック次第)

- **対象アプリのユーザー追加**: 任意のアプリ(Mattermost、Zoomチャット等)をbundle ID指定で追加できるUI
- **Web版のワークスペース別設定**: Web版でも「⌘Enter送信」宣言をワークスペース(ホスト)単位でできるようにする(今回はネイティブのみ)
- **Arcのフルパス取得改善**: 現状ドメインのみのため、DiscordのチャンネルURL等の厳密判定ができない
- **英語ローカライズ**: 海外ユーザー向け(Enter問題はCJK圏共通。韓国語・中国語IMEの検証も)
- **診断のワンクリック書き出し**: メニューから直近ログをファイル保存(サポート対応の効率化)
- **Homebrew cask掲載**: `brew install --cask unienter`(公証済みが事実上の前提)
- **Secure Input検知の表示**: パスワード入力中はタップにイベントが来ない仕様のため、状態をメニューに表示して「壊れた?」という誤解を防ぐ

## 検討中(やらない可能性もある)

- キーカスタマイズ(送信キーをCtrl+Enter等に変更) — 「設定を増やさない」方針と要相談
- メッセージ編集モード・スレッド等の文脈検知の精緻化
- 韓国語・中国語IMEの変換中判定(現状は日本語のみ厳密対応。他言語は素通し=安全側)
