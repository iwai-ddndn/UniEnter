日本語入力のために、作られています

この種のツールの最大の難所は「変換確定のEnter」。UniEnterはここに一番こだわりました。

🇯🇵

変換確定のEnterは、絶対に触らない

日本語IME(macOS標準・Google日本語入力)の変換中を推定し、確定のEnterは無加工で素通し。判定に迷ったときも「何もしない」に倒すフェイルセーフ設計です。

🌐

ブラウザのWeb版にも

Safari・Chrome・Arc などで開いた Slack / Teams / Discord / Chatwork のWeb版も同じ操作感。アドレスバー入力中は自動で干渉を止めます。

設定は「対象アプリのオン/オフ」と少しだけ。アプリ側で送信キーを⌘Enterに変えている場合も、それを宣言すれば共存できます。

よくある質問

なぜアクセシビリティ権限が必要?

キーイベントを書き換えるにはmacOSのイベントタップという仕組みを使う必要があり、これにアクセシビリティ権限が求められます。必要な権限はこれひとつだけで、初回起動時に画面の案内に沿って許可できます。

日本語の変換確定のEnterが送信されたりしない?

しないよう設計しています。変換中と推定される間のEnterは一切加工せずIMEに渡します。さらに判定できないケースでは常に「加工しない」側に倒すため、誤って確定が壊れる方向には動きません。

アプリ側で「⌘Enterで送信」に設定している

そのアプリは既に UniEnter と同じ挙動です。設定画面でそのアプリの「送信キー」を⌘Enterに切り替えると、UniEnterは書き換えを行わず、アプリ本来の動作をそのまま使います。

入力内容を読まれたりしない?

読みません。判定に使うのはEnter系のキーイベントと修飾キー、前面アプリ、入力ソース(日本語かどうか)だけです。ネットワーク通信は行わず、ログの外部送信もありません。

対象外のアプリへの影響は?

ゼロです。前面アプリが対象リストにない場合、イベントには一切手を触れません。パスワード入力(Secure Input)中はmacOSの仕組み上そもそもイベントが渡ってきません。
